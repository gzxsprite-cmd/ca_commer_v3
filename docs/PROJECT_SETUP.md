# PROJECT_SETUP

## Stage Definition
**Stage-0** is a Pages-demo foundation stage. The objective is to establish a stable, understandable product skeleton before deep business implementation.

## Stage-0 Scope
Included in this stage:
- Platform-level structure
- Two-workspace separation
- Role baseline (AM, PjM, CM, SCP, CA)
- Shared shell and top-level navigation baseline
- Lightweight UI and page grammar rules
- Documentation governance for future iterations

## What Stage-0 Should Achieve
- A common language for future development
- A consistent page and navigation baseline
- Clear boundaries for safe incremental work
- A clean path from demo pages to full application architecture

## Intentionally Deferred
- Detailed workflow and approval rule variants
- Final business fields and validations
- Final backend APIs and persistence model
- Final role permission matrix
- Performance/security hardening for production scale

## Success Definition
Stage-0 is successful when:
1. Team members can identify the platform structure quickly.
2. New pages can be added through a consistent shell + grammar approach.
3. Role-home and workspace identity are clear.
4. Future work can iterate page-by-page without redoing the foundation.
