# ROLE_AND_NAV_BASELINE

## Baseline Roles
- **AM**
- **PjM**
- **CM**
- **SCP**
- **CA**

Role definitions remain baseline-level in Stage-0; detailed responsibilities and field-level actions are intentionally deferred.

## Workspace Primary / Secondary Role Orientation
### Daily Execution Workspace
- Primary: **AM, CM, CA**
- Secondary: **PjM, SCP**

### Tracking & Planning Workspace
- Primary: **SCP, CA, PjM**
- Secondary: **AM, CM**

This is a navigation and emphasis baseline, not a final permission matrix.

## Top-Level Navigation Philosophy
- Keep top-level nav short and stable.
- First-class entries:
  - Role Home
  - Daily Execution Workspace
  - Tracking & Planning Workspace
- Use consistent naming; avoid synonym drift.
- Keep navigation depth shallow for demo stage.

## Baseline Role-Home Expectations
Each role home should provide:
1. Current focus items
2. Pending actions (with clear next step)
3. Quick links into relevant workspace pages
4. Light summary indicators (not heavy dashboards)

## What Is Not Included Yet
- Detailed workflow screens
- Final menu trees per role
- Final permission/authorization policies
